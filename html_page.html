HTML
Body, Head, and Title
  • <body>: everything in this element is shown inside the main browser window.
  • <head>: contains information about the page.
  • <title>: shown at the top of the browser.
Text
Structural Markups:
  • <h1>...<h6>: used for main headings, subheadings, so on and so forth.
  • <p>: creates a paragraph.
  • <b>: makes characters appear bold.
  • <i>: makes characters appear italic.
  • <sup>: superscript.
  • <sub>: subscript.
  • <br />: adds a line break.
  • <hr />: adds a horizontal rule between sections.
Semantic Markups:
  • <strong>: indicates that its content has strong importance.
  • <em>: indicates emphasis.
  • <blockquote>: used for longer quotes that take up an entire paragraph.
  • <q>: used for shorter quotes that sit within a paragraph.
  • <abbr>: for abbreviations or acronyms.
  • <cite>: indicates where the citation is from.
  • <dfn>: indicates the defining instance of a new term.
  • <address>: contains contact details for the author of the page.
  • <ins>: used to show content that has been inserted into a document.
  • <del>: shows text that has been deleted from it.
  • <s>: indicates something that is no longer accurate or relevant.
Extra Markup:
  • <DOCTYPE html>: tells browser which version of HTML the page is using.
  • <!-- comment goes here -->: comments.
  • id="...": id attribute.
  • class="...": class attribute.
  • <div>: grouping text & elements in a block.
  • <span>: grouping text & elements inline.
  • <iframe>: inline frame.
  • <header>: appears at the top of the page.
  • <footer>: appears at the bottom of the page.
  • <nav>: contains the major navigational blocks on the site.
  • <article>: a container for any section of a page that could stand alone.
  • <aside>: either contains information related to the article but not essential to its overall meaning, or contains content that is related to the entire page.
  • <section>: groups related content together.
  • <hgroup>: groups together a set of one or more h1 through h6.
Lists
Ordered lists:
  • <ol>: ordered list.
  • <li>: item list.
Unordered lists:
  • <ul>: unordered list.
  • <li>: item list.
Definition lists:
  • <dl>: definition list.
  • <dt>: contains the term being defined.
  • <dd>: contains the definition.
Links
Absolute URL
  • link to a different website
  • <a>: links which has an attribute called href
  • href="...": the page that you want people to go when they click on the link
Relative URL
  • links to other pages within the same site
  • <a>
  • href="...".html contains links from a page withing the same site
Relative Link Type
  • Same folder: to link to a file in the same folder, just use the file name.
  • Child folder: use the name of the child folder/then the file name.
  • Grandchild folder: use the name of the child folder/then the name of the granchild/then the file name.
  • Parent folder: use ../ to indicate the folder above the current one then follow it with the file name.
  • Grandparent folder: repeat ../ to indicate you want to go up two folders, then follow it with the file name.
Email Links
  • href="mailto:...": creates a link that starts up the user's emial program and addresses an email to a specified email address.
Opening new links to a new window
  • target="_blank": opens a new window.
Linking to a specific part of the same page
  • href="#idName"
Images
Adding Images
  • <img>: adds an image into the page.
  • src="...": tells the browser where it can find the image file.
  • alt="...": provides a text description of the image which describes the image if you cannot see it.
  • title="...": provides additional information about the image.
Figure and Figure Caption:
  • <figure>: contains images and their caption so that the two are associated.
  • <figcaption>: adds a caption to an image;
Tables
Basic Table Structure:
  • <table>: used to create a table. Contents of the table are written out row by row.
  • <tr>: indicates the start of each row
  • <td>: represents each cell of a table.
  • <th>: represents the heading for either a column or a row.
    • scope="col": indicates a heading for a column.
    • scope="row": indicates a heading for a row.
Spanning Columns:
  • colspan="num"
Spanning Rows:
  • rowspan="num"
Long Tables:
  • <thead>: headings of the table.
  • <tbody>: body of the table.
  • <tfoot>: footer of the table.
Forms
Form Structure:
  • <form>: form controls.
  • action: attribute where its value is the URL for the page on the server that will receive the information in the form when it is submitted.
  • method="get": the values from the form are added to the end of the URL specified in the action attribute.
  • method="post": the values are sent in HTTP headers.
Text input:
  • <input>: creates several different form controls. The type attribute determines the kind of input.
  • type="text": creates a single-line text input.
  • name: the value of this attribute identifies the form control.
  • maxlength: limits the number of characters a user may enter into the text field.
Password input:
  • <input>
  • type="password": creates a text box that acts just like a single-line text input, except the characters are blocked out. They are hidden.
  • name, maxlength
Text area:
  • <textarea>: creates a multi-line text input.
Radio button:
  • <input>
  • name: generally should be the same for all of the radio buttons used to answer that questions.
  • value: indicates that value is sent to the server for the selected option
  • checked: can be used to indicate which value should be selected when the page loads. Only one should use this attribute.
Checkbox:
  • <input>
  • type="checkbox"
  • name, value, checked
Drop Down List Box:
  • <select>: used to create a drop down list box.
  • name
  • <option>: used to specify the options that the user can select from.
  • value
  • selected: indicate the option that should be selected when the page loads.
Multiple select box:
  • <select>
  • size
  • multiple: allows user to slect multiple options from the list.
File input box:
  • <input>
  • type="file": creates abox that looks like a text input followed by a browse button.
Submit Button:
  • <input>
  • type="submit": used to send a form to the server.
  • name, value.
Image Button:
  • <input>
  • type="image"
  • src
Button:
  • <button>
  • type="hidden": form controls are not shown on the page.
Labelling form controls:
  • <label>
  • for: states which form control the label belongs to.
Grouping Form Elements:
  • <fieldset>: groups related form controls together.
  • <legend>: helps identify the purpose of that group of form controls.
Date Input:
  • <input>
  • type="date"
Email & URL Input:
  • <input>
  • type="email"
  • type="url": when you are asking a user for a web page address.
Search Input:
  • <input>
  • type="search": creates a single line text box for search queries.
  • placeholder: its value is text that will be shown in the text box until the user clicks in that area.
Flash, Video & Audio
  • <video>: allows you to control video playback.
  • src
  • poster: allows you to specify an image to show while the video is downloading.
  • controls: indicates that the browser should supply its own controls for playback.
  • autoplay: specifies that the file should play automatically.
  • loop: indicates that the video should start playing again once it has ended.
  • preload: tells the browser what to do when the page loads.
  • none: should not load the video until the user presses play.
  • auto: should download the video when the page loads.
  • metadata: collect information such as the size, first frame, track list, and duration.
Multiple video sources:
  • <source>: specifies the location of the file to be played.
  • src, type
  • codecs: codec that is used to encode the video.
Audio:
  • <audio>: includes audio files in your pages.
  • src
  • controls, autoplay, preload, loop.
Multiple Audio Sources:
  • <source>
  • src, type.
